vertical-align详解(1)

背景知识

行内元素基本术语

  • 匿名文本(anonymous text)：所有未包含在行内元素的字符串，空格也是匿名文本的一部分；
  • em框：em框在字体中定义，也成为字符框(character box)，实际的字形可能比它的em框更高或者更矮，font-size = em-height
  • 内容区：非替换元素中，内容区是元素中各字符的em框串在一起构成的框。替换元素中，内容区就是元素的固有高度加上可能存在的margin,border, padding；
  • 行间距(leading)：是font-size和line-height的差值，这个值分为2部分，分别应用到内容区的顶部和底部，这2部分成为半间距(half-leading)，行间距只应用于非替换元素，行间距也可以为负数；
  • 行内框：这个框通过向内容区增加行间距来描述。对于非替换元素，行内框的告诉刚好等于line-height的值。对于替换元素，行内框的高度恰好等于内容区的高度，因为行间距不应用到替换元素；
  • 行框：包含该行中出现的行内框的最高点和最低点的最小框，也就是最高行内框的上边界到最低行内框的下边界。

    如图em框，font-size = em-height

有用的概念

  • 内容区类似块级元素的内容框；
  • 行内元素的背景应用于内容区及其内边距；
  • 行内元素的边框要包围内容区及所有内边距和边框；
  • 非替换元素的内边距，边框，外边距对行内元素没有垂直效果，他们不会影响元素行内框的高度；
  • 替换元素的外边距和边框会影响该元素行内框的高度。

参考资料：css权威指南

字体常识

  • baseline: 基线，即a,c,z,x等小写字母的底边线，如图中3
  • ex = x-height = baseline - mean-line ，简而言之就是小写英文字符x的高度，如图中10

vertical-align属性详解

baseline:
默认的对齐样式，将元素行内框的基线和包含该元素的行框的基线对齐。如果元素没有基线，将元素margin-bottom的底部和包含该元素的行框的基线对齐。

引用自www.w3.org资料
这段的意思是，一个inline-block的元素的基线是它在在正常文档流里的最后一个inline元素决定，除非它没有正常文档流的行内元素，或者它的overflow属性不是visible，这种情况下，它的基线是margin-bottom的底部。
这种情况会在下次进行讨论。

top:
将元素行内框的顶端与包含该元素的行框的顶端对齐。

bottom:
将元素行内框的底端与包含该元素的行框的底端对齐。

text-top:
将元素行内框的顶端与父元素内容区的顶端对齐。

text-bottom:
将元素行内框的底端与父元素内容区的底端对齐。

middle:
将元素行内框的垂直中点与父元素基线上0.5ex处的一点对齐。

super:
将元素的内容区和行内框上移，上移动的距离由用户代理决定。

sub:
和super相同，只是动作为下移。

百分比(persentage %):
百分比数值相对于当前元素的line-height计算，正百分比会使元素的基线相对于父元素的基线升高，负百分比会降低，例如line-height为20px的元素，vertical-align: 50%,将会使其基线升高10px.

数值:
类似于计算后的百分比值, 0 即 baseline.
